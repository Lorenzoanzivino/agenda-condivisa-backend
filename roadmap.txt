FASE 1: Autenticazione e Refactoring Utenti (Backend + Frontend)

    [ ] Backend (API Gateway / User-Service):

        Rimuovere la vecchia logica di registrazione manuale (form con password, data di nascita, ecc.).

        Integrare Google OAuth2 (Spring Security).

        Creare l'endpoint per il primo accesso: l'utente loggato con Google fornisce solo il "Nome di visualizzazione" (es. "Lorenzo").

    [ ] Frontend (Mobile):

        Creare una schermata di Login minimale con un unico tasto "Accedi con Google" in stile Glassmorfismo.

        Creare un piccolo form post-login (solo per i nuovi utenti) per inserire il nome.

FASE 2: Motore Calendario e Condivisione (Backend)

    [ ] Calendar-Service (Entità e DTO):

        Aggiornare il modello Evento per includere il flag booleano tuttoIlGiorno (all-day).

        Assicurarsi che i campi siano: titolo (obbligatorio), descrizione (opzionale), dataInizio, dataFine, tuttoIlGiorno.

    [ ] Sistema di Condivisione (Logica "Join by Code"):

        Creare entità CalendarioCondiviso.

        Endpoint per generare un codice alfanumerico univoco (es. JD7F-9K2A).

        Endpoint per "Unisciti al calendario" tramite inserimento codice.

    [ ] Query di separazione:

        Logica per recuperare gli eventi di oggi separati in due flussi: "Eventi Privati" ed "Eventi Condivisi". (Nessun avviso di conflitto tra i due).

FASE 3: UX/UI e Interattività (Frontend)

    [ ] Dashboard Odierna (HomeScreen):

        Implementare lo Swipeable Pager (Carousel) orizzontale.

        Pagina 1: Eventi Privati. Indicatori a pallini (primo acceso). Atmosfera colori base (viola profondo).

        Pagina 2: Eventi Condivisi. Indicatori a pallini (secondo acceso). Cambio di atmosfera dinamico (es. tonalità più calde o blu notte) tramite Glassmorfismo.

    [ ] Calendario Mensile (Navbar):

        Aggiungere la Bottom Navbar con le icone (Home, Calendario, Condivisione).

        Sviluppare la griglia mensile classica a 30/31 giorni.

    [ ] Gestione Eventi (CRUD):

        Cliccando su un giorno (dal mese) o dal tasto "+" (nella Home), aprire un modale di creazione/modifica.

        Form: Titolo, Descrizione, Data/Ora (o spunta "Tutto il giorno").

        Tasti azione: Salva, Modifica, Elimina.

FASE 4: Notifiche Asincrone

    [ ] RabbitMQ + Notification-Service:

        Configurare l'exchange per reagire ad azioni specifiche (es. "Nuovo evento aggiunto al calendario condiviso").

        Preparare i log/alert lato Notification-Service. (Agent-Service rimane in sospeso).

FASE 5: Infrastruttura e Deployment (Cloud)

    [ ] Setup Oracle Cloud Free Tier:

        Creare e configurare la Virtual Machine (Architettura ARM64, 24GB RAM).

        Aprire le porte di rete necessarie nel firewall Oracle.

    [ ] Containerizzazione Multi-Architettura:

        Compilare le immagini Docker di Spring Boot specificando la piattaforma ARM (--platform linux/arm64) per garantirne il funzionamento su Oracle.

    [ ] Deployment:

        Installare Docker e Docker Compose sulla VM Oracle.

        Creare un tunnel (es. Cloudflare) per un indirizzo HTTPS fisso.

        Lanciare docker compose up -d sul server di produzione.
